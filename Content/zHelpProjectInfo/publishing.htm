Publishing the help project

The TOC is the 'IFTDSS Online Output (Master)

The publishing Target is in the Online Output folder, 'HTML5 - Top Navigation'

Pushing the project to Github for IBM

In the target, go to the Publishing section of the target, select New Destination, then specify the desired folder on your local machine that you would like to store a copy of the output file.

After the project is Built, select Publish to make a copy of the finished help project in your local folder that is linked to Github (https://github.com/hyde1520/iftoutput)

Publishing the project to keep our working version current

To update the project:

Begin by saving your project.

Next, go to Source Control and select Commit All.

You will be prompted to add a sentence to summarize the updates to the project, then select Commit.

Next, select Push to send the local files to the Github repository. (https://github.com/photuris5/V3iftdss)

A box will appear to confirm the location to push the files to. Leave the default source as 'origin' and select OK.

You will see a brief progress bar as the files are pushed to the remote server.
