Inputs Tab

The Inputs tab in the "Run:" dialog box is where you define or setup your FlamMap run. Here you enter the non-spatial inputs and control how the fire behavior characteristics are calculated.

You should first name your run to avoid too many New Run items in your "Tree" pane.  

For each ASCII text file input there are two buttons to the right of the text box available to help you.

This button opens the standard Windows "Open" dialog box where you can navigate and select the correct ASCII text file.

After a file is selected the pencil button opens the simple FlamMap text editor, similar to Notepad, where you can view, edit, and save the file.

Fuel Moisture Files

Initial Fuel Moisture (.FMS) File

The only required ASCII text file input for a FlamMap run is the Initial Fuel Moisture (.FMS) File. This file is used for all three of the Fuel Moisture options described below.

To load a Initial Fuel Moisture (.FMS) File for your FlamMap run, click the button to display the "Open" dialog box. Navigate to the appropriate folder, select the desired Initial Fuel Moisture (.FMS) File and click the Open button.

By clicking the button you can inspect the Initial Fuel Moisture (.FMS) File and modify the fuel moistures for individual fuel models using the FlamMap Text Editor.

You can also create a new Initial Fuel Moisture (.FMS) File by clicking the button.

Custom Fuel Model (.FMD) File  

If custom fuel models were included in the Fuel Model Grid used in the Landscape (.LCP) File, the Custom Fuel Model (.FMD) File needs to be include in the run. Select the Use Custom Fuels (*.fmd) checkbox and click the button to display the "Open" dialog box. Navigate to the appropriate folder, select the desired Custom Fuel Model (.FMD) File and click the Open button.

Winds

FlamMap uses a 20 foot wind speed and direction at one point in time to calculate fire behavior characteristics. Winds are automatically adjusted to midflame speed where appropriate using the individual cell canopy characteristics.  There are five choices to provide the 20 foot wind speed and direction input.

NOTE: Wind (.WND) Files are only used in FlamMap for the cloud cover values when using the Fuel Moisture Conditioning option.

Single Wind Direction

Selecting the Wind Direction radio button uses the same wind direction for every cell in the analysis area. Select a specified wind direction with the Azimuth (Degrees) spin box and a 20 foot wind speed in the Wind Speed (MPH@20') spin box.

Uphill/Downhill Winds

Selecting the Wind Blowing Uphill/Downhill radio buttons utilizes the upslope/downslope direction from the aspect theme of the Landscape (.LCP) File for each grid cell to calculate the fire behavior characteristics. Flat cells with no aspect use the direction from the Azimuth (Degrees) spin box. Selecting a 20 foot wind speed in the Wind Speed (MPH@20') spin box completes the inputs for this option.

This option would be appropriate for modeling the fire behavior with simple diurnal winds. Uphill winds can also be used to calculate a near-maximum fire behavior characteristics. Remember that complex terrain winds, such as funneling in a narrow canyon, are not modeled with this option. You need to use the Generate Gridded Wind or Gridded Wind Files options to model fire behavior with complex terrain induced winds.

Generate Gridded Wind

One method of modeling fire behavior using complex wind and terrain interactions can be done using the Generate Gridded Wind option. Selecting this radio button activates the WindNinja Options button, clicking which will open the "WindNinja Options" dialog box. Details for using the WindNinja Options are found in the Wind Vectors topic.

Gridded Wind Files

Modeling fire behavior using complex wind and terrain interactions can also be done using wind grids created outside of FlamMap. Two wind grids, speed and direction, need to be generated by computational fluid dynamics or weather modeling software and saved as ASCII grid format files before you can use this option.

To use wind grids for the Winds input select the Wind Grids radio button and select the Direction and Speed grid files using the respective buttons.  

Canopy Characteristics

If the canopy characteristics are specified in the currently loaded landscape then those values are used in the run calculations and the Height(m), Canopy Base Height(m), and Canopy Bulk Density(Kg/m3) spin boxes are dimmed and not available.

A foliar moisture content for conifer crown fuels is always required in the Foliar Moisture Content (%) spin box. Usually this should range between 80 and 130 percent, with 120% being appropriate for average mid-summer conditions and 100% a good value for drought conditions.  

If any or all of the three optional canopy characteristic themes (stand height, crown base height, and crown bulk density) are not included in the Landscape (.LCP) File and you wish to model crown fire characteristics you can specify the canopy characteristics globally (a constant for the entire landscape). Use the spin boxes to specify a value for the Height(m), Canopy Base Height(m), and Canopy Bulk Density(Kg/m3) canopy characteristics.

Canopy characteristics (either from the Landscape Themes or constants) are ignored for fire behavior calculations where the canopy cover is zero or blank since FlamMap assumes there is no tree overstory.  

Fuel Moisture Settings

There are three options for the dead fuel moistures used in calculating the fire behavior characteristics in FlamMap.

The simple option is to use fuel moistures from the Initial Fuel Moisture (.FMS) File specified above by selecting the Use Fixed Fuel Moistures from Fuel Moisture File radio button. This option uses the same fuel moistures for all grid cells of a fuel model across the landscape regardless of topography or shading. The Fuel Moisture Output Grids are not available when this fuel moisture option is selected.

Selecting either the Use WTR and WND Files or Use Weather Stream radio buttons calculates separate dead fuel moistures for each cell based on the topography, shading, weather, and conditioning period length. When selecting these two options text files with weather data are required. For the Use WTR and WND Files option you must specify a Weather (.WTR) File, Wind (.WND) File, and the start and end times of the conditioning period. Selecting the Use Weather Stream radio button requires a Weather Stream (.WXS) File and the start and end times of the conditioning period.

The dead fuel moistures at the end of the conditioning period are used in calculating the output grids so choose a conditioning period, especially the end day, with weather appropriate to your analysis. The conditioning period must utilize dates included in the Weather (.WTR)/Wind (.WND) Files or Weather Stream (.WXS.) File. Conditioning periods need only be a maximum of 3-5 days. Live fuel or foliar moistures are not modified when using the Fuel Moisture Conditioning options. Fuel moisture conditioning can be used when calculating any of the FlamMap products; Fire Behavior Outputs, MTT fire growth, Burn Probabilities, or Treatment Optimization.

Buttons

The buttons near the bottom of the "Run:" dialog box control calculating the FlamMap run and saving the run settings.

Clicking the OK button saves all the current run settings and button status, then closes the "Run:" dialog box without calculating the run. The Cancel button closes the "Run:" dialog box without saving, the run settings remain the same as when you last opened the "Run:" dialog box.

The Launch and Apply buttons are dimmed when a new "Run:" dialog box is first opened. Once any changes are made in the "Run:" dialog box the Apply button becomes active. Clicking the Apply button applies (or loads) the settings, if the proper settings from the Inputs and Outputs tabs have been applied, the Launch button then becomes active. If all the proper settings have not been applied the Launch button remains dimmed.

Clicking the Launch button begins calculating the FlamMap run and opens this progression box.

If both the Launch and Apply buttons are both active it indicates changes have been made to the run that have not been applied. The Launch button will begin the run but use settings from when the Apply button was last clicked, not the current settings shown in the "Run:" dialog box.  

Status Bar

The status bar at the bottom (below the buttons) of the "Run:" dialog box informs you of the status of your run. It is divided into three sections:

  1. the inputs box which informs you if you have all the necessary inputs selected for the Run,

  2. the second status box displays the number of output themes you have selected,

  3. the third box lets you know if any previously calculated outputs are still valid for the conditions currently set on the Inputs Tab.

Any changes you make to the Inputs or Output Tabs are not reflected on the status bar until you click the Apply button.
