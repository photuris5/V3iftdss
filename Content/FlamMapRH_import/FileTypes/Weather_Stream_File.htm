Weather Stream File (.WXS)

The Weather Stream (.WXS) File is an optional ASCII text file required to utilize the dead fuel moisture model in FlamMap. This file is the equivalent of the Weather (.WTR) and Wind (.WND) Files combined. This format also has the advantage of providing the actual periodic temperature and humidity values for the dead fuel moisture model. This becomes important if the max/min values in a Weather (.WTR) File do not follow the typical pattern of afternoon max temperatures and early morning low temperatures.

A Weather Stream (.WXS) File contains periodic, usually hourly, observations on temperature, humidity, precipitation, wind speed, wind direction, and cloud cover that depicts a temporal weather stream. The weather stream greatly oversimplifies actual variation in weather. However, this format is an attempt to limit to a practical level, the amount of weather information for modeling the variation in dead fuel moistures due to topography and shading.

Each Weather Stream (.WXS) File must contain data in the space delimited ASCII format specified below. There are a variety of ways to create or edit Weather Stream (.WXS) Files,

  • a text editor such as Notepad or WordPad,
  • a spreadsheet application and save to a space delimited file,
  • the FlamMap Custom Editor (accessed with the buttons on the Inputs tab of the "Run:" dialog box),
  • export from FireFamily Plus.

Example of .WXS File

Header lines in the Weather Stream (.WXS) File consist of an attribute name and colon followed by the value,

  • RAWS_ELEVATION: Elevation is in feet or meters above sea level. NOTE: these units (feet or meters) do not have to be the same as the landscape elevation theme (integer) but are in the units specified by RAWS_UNITS:,
  • RAWS_WINDS: an optional informational line added in FireFamilyPlus exports,
  • RAWS_UNITS: the units are selected by inserting the word ENGLISH or METRIC.

Column name row is optional. All inputs must be integers. Each column must be space delimited (leave a space between columns).

  • Year is the year of the observation,
  • Mth is the month of the observation,
  • Day is the day of the observation,
  • Time  is the time of the observation, 0-2359, to the nearest minute (integer),
  • Temp is the temperature at observation time in degrees Fahrenheit or Celsius (integer).
  • RH is the relative humidity at observation time in percent, 0 to 99 (integer).
  • HrlyPcp is the precipitation amount for the time period specified in hundredths of an inch or millimeters (integer),
  • WindSpd is the wind speed at observation time, either the 20ft wind speed specified in miles per hour or the 10m wind speed in kilometers per hour (integer),
  • WindDir is the direction the wind is from at observation time in degrees, clockwise from north (0-360), (integer),
  • CloudCov is cloud cover specified as a percentage, 0 to 100 (integer).

Using the Custom Weather Stream File Editor

The Weather Stream (.WXS) File Custom Editor is accessed with the button on the Inputs tab of the "Run:" dialog box.

Editing an Existing Weather Stream File

If you currently have a Weather Stream (.WXS) File loaded, this dialog box will allow you to edit the contents of that file. The Date field cannot be edited, nor can observations be created or deleted. You can only modify existing observations, Units, or Elevation.

You can load a new Weather Stream (.WXS) File into the editor with the Open File button, the new file will be loaded in the Inputs tab of the "Run:" dialog box when you exit the editor.

The Save As button opens a "Save As" dialog where you can save the current observations as a new Weather Stream (.WXS) File.

The Inspector button will check the current edits for validity.

The Help button will bring you to the Using the Custom Weather Stream File Editor heading above in this online help system.

 If any changes are made with the editor you will be asked if you want to save these changes when you exit the editor by clicking the Close button.
