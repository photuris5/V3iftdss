Data Requirements

Before beginning a FlamMap project it is important to spend some time to identify the project data needs and the data sources. FlamMap utilizes the same data as FARSITE, so if you have the data to do FARSITE fire simulations for a project, you are all set to use FlamMap.

FlamMap requires the support of a Geographic Information System (GIS) to generate, manage, and provide spatial data themes containing fuels, vegetation, and topography. Five raster data themes are required to run FlamMap; these include elevation, slope, aspect, fuel model, and canopy cover and must be downloaded as GRASS ASCII or ARC GRID ASCII format and combined to a single Landscape (.LCP) or GeoTIFF (.TIF) File. (see File > Generate Landscape). All themes must be:

  • co-registered (e.g. have the same reference point and units)
  • identical resolution (e.g. cell size must be the same for all themes)
  • same extent (the corners of the rectangular spatial region must be the same)
  • same projection and datum (the methods to show a curved world as a flat surface)

There is one non-raster text file required to run FlamMap, the Initial Fuel Moistures (.FMS) File.

Some GIS data themes are described as "optional". This means only that you do not need them to run a surface fire simulation.  It DOES NOT mean that these are not required to calculate some aspects of fire behavior (e.g. crown fire) that are important to your analysis.

To utilize the dead fuel moisture model in FlamMap you also need text files of weather and wind data. Weather (.WTR) and Wind (.WND) Files are the same needed to run FARSITE. FlamMap also has the option of using a Weather Stream File (.WXS) for the weather and wind data.
