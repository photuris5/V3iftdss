Wind Grid Files (.ASC)

FlamMap can use gridded inputs for the speed and direction wind variables. These two grids need to be generated by external computational fluid dynamics or weather modeling software and saved as ASCII grid format files before you can use this option. You can also use the Wind Ninja Option in FlamMap to generate the grids in a previous run.

Data Units

Data units are the same as those used for other wind stream input options in FlamMap, however metric units inputs are not supported at this time:

  • Wind Speed MPH, 20 foot open wind (integer)
  • Wind Direction Azimuth Degrees (integer)

Wind adjustment factors (WAF) are calculated the same for gridded inputs as for stream inputs. The exact method is described with equation #44 in Finney (1998).

Resolution and Registration

Each of the two gridded inputs can be of different resolutions and do not need to be co-registered. For example, the resolution for speed could be 500m but the wind direction resolution could be 50m. The coverage of each file can also be offset so long as the extent of each grid file covers the analysis area of interest.

Input Formats

ASCII Grid files must have the following six line header included in each file:

 

File Line

Comment

NCOLS xxxx

Replace xxxx with the number of grid columns.

NROWS yyyy

Replace yyyy with the number of grid rows.

XLLCORNER xxxxxxx

Replace xxxxxx with the lower left grid X-coordinate

YLLCORNER yyyyyy

Replace yyyyyy with the lower left grid Y-coordinate

CELLSIZE zz

Replace zz with the grid cell size

NODATA_VALUE -99999

use -99999 for no data cells when generating your grid.

The column data values are separated by a space and the end of a row identified by an end-of-line character.
