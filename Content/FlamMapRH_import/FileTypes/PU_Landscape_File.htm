Landscapes

The landscape is the spatial data needed to run FlamMap. These must include five basic rasterized data themes imported from a GIS;

  • elevation,
  • slope,
  • aspect,
  • fuel model,
  • canopy cover.

The landscape may also include optional files for;

  • stand height,
  • crown base height,
  • canopy bulk density,
  • coarse woody fuels (not used by FlamMap),
  • duff loading (not used by FlamMap).

The optional parameters that are not provided by a raster file need be set as constants across the entire landscape in the "Inputs" tab of the "Run:" dialog box.

Landscapes can be loaded and/or saved in two formats, a Landscape (.LCP) File or a GeoTIFF (.TIF) format file.

Landscape File (.LCP) Format

An Landscape (.LCP) File is a binary file comprised of a header and a body of short integers for each of the themes it contains. The header contains information on the bounds of the area, the resolution of the cells, and the units of the themes. An optional Projection (.PRJ) File may be present in the same folder.

This is the same Landscape (.LCP) File format used in FARSITE v4.x so Landscape (.LCP) Files generated in either application are interchangeable.

You can generate a Landscape (.LCP) File in FlamMap using the File > Create Landscape (.LCP) File... command which requires you to specify each of the ASCII grid files and their units.. A Landscape (.LCP) File can also be generated using the Input > Landscape Utilities > Generate LandscapeFile command in FARSITE v.4.

Within the United States, pre-made Landscape (.LCP) Files with a valid Projection (.PRJ) File can be found at www.landfire.gov.

If you have an existing FARSITE Landscape (.LCP) File make sure it is the version 4 format. Landscape (.LCP) Files created in FARSITE v3.x can be updated in FARSITE v4.x with the Input > Landscape Utilities > Convert Landscape File version 3 to 4 command.

GeoTIFF File (.TIF) Format

FlamMap can also save a landscape as a GeoTIFF (.TIF) format file. Valid projection information is required to save in this format. At this time, a standard Landscape (.LCP) File has to be created first and then converted to a GeoTIFF. These usage and limitations apply:

  • Creates a multi-band (10 Band) GeoTIFF.
  • Requires a valid landscape and projection file.
  • Allows for the creation of landscapes in a GeoTIFF File format compatible for use in WindNinja(Ver. 2.1.0) and ArcMap 10.0.
  • GeoTIFF  landscape files are currently not usable in FARSITE.

There are two ways to create a landscape in a GeoTIFF File format. If a Landscape (.LCP) File is already loaded in a FlamMap Project the shortcut menu from the tree pane is the easiest, as long as a valid Projection (.PRJ) File is in the same folder as the loaded Landscape (.LCP) File. Otherwise the File > Convert LCP to GeoTIFF command must be used.

From the Tree Pane

A New Analysis Area or Entire Landscape (Default)) item can be saved in a GeoTIFF (.TIF) format file by right-clicking the item and selecting the Create Landscape command from the shortcut menu. The command opens a "Select Destination File" dialog box allowing the user to navigate to the location they want to store the newly created GeoTIFF File. The user must designate Save as Type: as GeoTIFF files(*.tif). A valid Projection (.PRJ) File, with the same name as the Landscape (.LCP) File, must be in the same folder as the loaded Landscape (.LCP) File.

From the Menu Bar

If the Landscape (.LCP) File is not loaded or the Projection (.PRJ) File does not have the correct name/folder select File > Convert LCP to GeoTIFF from the Menu Bar. This will open the "Convert LCP to GeoTIFF" dialog box and the user will need to supply the Source LCP File, Source Projection File, and Destination GeoTIFF File.

Click the button to the right of the Source LCP File text box to display an "Select Source LCP File" dialog box. Navigate to the appropriate folder, select the desired Landscape (.LCP) File and click the Open button.

If no projection file is available in the same location as the Source LCP File the following error box will appear.

Click OK in the error message box will close it. Load a Projection (.PRJ) File by clicking the button to display a “Select Source Projection File” dialog box. Navigate to the correct projection file; make sure the file name is displayed in the File Name text box and click Open.

Select the Destination GeoTIFF File location by clicking the button and name the new file, click the Save button.

Finally click the OK button on the "Convert LCP to GeoTIFF" dialog box to finish the conversion.

This option has the flexibility of being able to navigate to a different location for the projection file if it is not located in the same directory as the Landscape (.LCP) File.

NOTE: Navigating to and selecting a Projection (.PRJ) File is not the same as reprojecting (transforming a dataset from one projection to another) this must occur outside FlamMap in a GIS system. The selected Projection (.PRJ) File MUST represent the map datum and coordinate system of the Source LCP File.
