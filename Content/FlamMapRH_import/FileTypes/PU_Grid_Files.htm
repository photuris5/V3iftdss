Grid Themes

Grid or raster refers to a type of spatial data used by a GIS that consists of rows and columns of rectangular cells. Each cell contains values or attributes for that particular theme or layer. For example, if the theme is elevation, then the attributes will contain elevations in meters or feet. FlamMap uses raster data as inputs for all spatial data on topography, fuels, and crown structure. FlamMap can also export fire behavior information as raster themes for use in a GIS.

 

 

FlamMap can import only ARC GRID ASCII format raster files.

Note: GRASS raster files can be used to create a Landscape (.LCP) File in FARSITE v4. Then that Landscape (.LCP) File can be used in FlamMap.

All raster files used to create a Landscape (.LCP) File must be of identical resolution, projection, and must overlap exactly. Be sure that the values in each raster map have the correct units or correspond to the requirements for each theme. Auxiliary grid files do not have to be the same resolution or extent, but they should have the same projection and datum.  

All ARC GRID ASCII format raster files must have a header to be imported into FlamMap. If you are unsure, examine the first 6 lines of one of your raster file. You should see the following format:

ncols

610

nrows

545

xllcorner

372775.121594

yllcorner

5070054.003522

cellsize

30

NODATA_value

-9999

 

A procedure for building the raster files using ArcView can be found in this help system.

 

All of the raster files FlamMap exports are ASCII format, not binary.
