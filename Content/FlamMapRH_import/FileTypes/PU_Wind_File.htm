Wind File (.WND)

The Wind (.WND) File is an optional ASCII text file required to utilize the optional dead fuel moisture model in FlamMap. For convenience the Wind (.WND) File is the same format used in FARSITE. However, only the cloud cover data is used in the dead fuel moisture model. The wind speed and direction data are not used in FlamMap (the speed and direction are constants set in the Winds frame in the Inputs tab of the "Run:" dialog box). So if you are building a Wind (.WND) File to use exclusively in FlamMap you can fill in the wind speed and direction with zeros.

Wind information must be input as a stream of data contained in a Wind (.WND) File. Each Wind (.WND) File must contain data in the space delimited ASCII format specified below. There are a variety of ways to create or edit Wind (.WND) Files,

  • a text editor such as Notepad or WordPad,
  • a spreadsheet application and save to a space delimited file,
  • the FlamMap Text Editor (accessed with the buttons on the Inputs tab of the "Run:" dialog box),
  • Custom Editors for each ASCII text file type are available in FARSITE,
  • generate using the Input > Generate from Types (WTR/WND) command in FARSITE,
  • generate in FireFamily Plus.

All inputs MUST be integers. Each column must be space delimited (leave a space between columns). FARSITE and FlamMap allow weather inputs in English or metric units. The units are selected by inserting the word ENGLISH or METRIC as the first line of the Wind (.WND) File.

Month Day Hour Speed Direction CloudCover

  • Hour is specified as 0-2359, to the nearest minute (integer).

  • Speed is either the 20ft wind speed specified in miles per hour or the 10m wind speed in kilometers per hour (integer)

  • Direction is specified in degrees, clockwise from north (0-360), (integer).

  • CloudCover is specified as a percentage, 0 to 100 (integer).

Note: Wind observations do not have to be on a regular interval. You can, for example, enter wind observations every 10 minutes during the afternoon, and only every 2 hours at night.

Using the FlamMap Text Editor

The pencil button, found to the right of the Wind File (*.wnd) text box on the Inputs tab of the "Run:" dialog box, opens the Wind (.WND) File named in the associated text box.

Generating Wind Files in Fire Family Plus

Wind (.WND) Files can be generated from hourly WIMS data in FireFamily Plus. From the menu bar in FireFamily Plus go to the Weather > Hourly Data Analysis > FarSite Exports command. Make sure you have a FireFamily Plus database that contains hourly observations.

Wind Grids

Wind in FlamMap can also be depicted using previously created Wind Grid Files or using the WindNinja option in FlamMap.
