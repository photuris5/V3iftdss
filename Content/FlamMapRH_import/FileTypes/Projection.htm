Projections

Projections are the information used to orient points or features from a grid or vector file on the surface of the earth. Since the earth is round and maps (computer screens, too) are flat various ways have been devised to "project" the earth's curved surface to a flat display.

Projection information for FlamMap is saved/transferred in two basic ways, embedded within the points file and as a separate file.

Embedded

GeoTIFF (.TIF) and Keyhole Markup Language (.KMZ & .KML) files are examples of where the projection information is stored with the feature information in one file.

Separate

Shapefiles (.SHP), Landscape (.LCP), and Grid (.ASC) files are are examples of where the projection information is stored in a separate Projection (.PRJ) File. Typically the Projection (.PRJ) File has the same name and location as the feature file and is differentiated by the .PRJ extension.

Projection Use in FlamMap

Projection information is not required to use FlamMap, a Landscape (.LCP) File without a Projection (.PRJ) File can be loaded. However there is always a projection for a Landscape (.LCP) File - FlamMap just doesn't have to know what it is.   So outputs can be created, viewed, and saved - they will have the same projection as the loaded Landscape (.LCP) File. Auxiliary themes without a Projection (.PRJ) File can also be loaded and displayed as long as the user makes sure they were created in the same projection as the loaded Landscape (.LCP) File.

WARNING: All geospatial inputs used in FlamMap are still required to be in the same map datum and coordinate system as FlamMap will not reproject features "on the fly" as most GIS applications do.

FlamMap will not create GeoTIFF (.TIF) or KMZ format outputs or attach a Projection (.PRJ) File to saved outputs without valid projection information.

A Valid Projection

The projection for a FlamMap Project is set by the projection of the landscape. Valid projection information can be linked to a FlamMap Project in two ways;

  • By loading a Landscape (.LCP) File with a Projection (.PRJ) File in the same folder, with the same name.
  • By loading a GeoTIFF (.TIF) format landscape, which has the projection information embedded in the .TIF file.
