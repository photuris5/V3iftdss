Minimum Travel Time

The minimum travel time (MTT) feature is a two-dimensional fire growth model.  It calculates fire growth and behavior by searching for the set of pathways with minimum fire spread times from point, line or polygon ignition sources. MTT has a wide variety of uses, in this lesson you will learn how to use MTT to simulate a fire from a point source ignition under a constant set of wind and fuel moisture conditions. A separate lesson using the MTT Burn Probabilities feature is also available, it is not necessary to complete this lesson before doing the Burn Probability Lesson.

The Minimum Travel Time (MTT) and Treatment Optimization features in FlamMap v. 3 can be very computer intensive. However this tutorial is designed to work in a reasonable time on a computer with the recommended requirements found on the FlamMap website. (Pentium 4 processor, 256 MB RAM, and Windows 2000 operating system)

The data files for this tutorial are found in your ...FlamMap3\Tutorial\Schley\ folder (directory).

When you first start FlamMap it opens a default project window named "FlamMap1". If you have been working on other FlamMap projects, create an new empty project window with the File > New command. It will have a name of "FlamMap2", "FlamMap3", etc.

Load a Landscape (.LCP) File

The first step is to load a Landscape (.LCP) File into the empty project; right-clicking on the ?No Landscape File item brings up a shortcut menu with just the Open and Help commands.

Select the Open command from the shortcut menu to open a standard Window's "Open" dialog box. Navigate to the ...FlamMap3\Tutorial\Schley\ folder and select schley_pre.LCP as your Landscape (.LCP) File and click the Open button. Your project window will look like this. To match the fuel model colors used in this tutorial select the Standard13 predefined legend.

The Inputs Tab

For an MTT run using the Inputs Tab is identical to other FlamMap runs. Like all FlamMap runs you can utilize wind grids and conditioning periods, but this lesson will use simple wind and weather inputs. Fill in the Inputs tab as shown below.

It is important to emphasize one of the major FlamMap assumptions at this point; all wind and fuel moisture conditions are constant. When a conditioning period is not utilized they do not even vary by aspect, elevation, or shading. In this run FlamMap will use the fuel moisture values in the Fuel Moisture (.fms) File for all calculations.

Make sure to click the Apply button before moving on to the next step.

The Minimum Travel Time Tab

Next select the Minimum Travel Time tab in the "Run:" dialog box.

Ignitions

For an MTT ignition you can use an existing shapefile made elsewhere or create a shapefile in FlamMap. For this run we will use an existing shapefile named start2.shp. Make sure the From File radio button is selected and click the button. In the "Open" dialog box navigate to ...FlamMap3\Tutorial\Schley\ folder and select the start2.shp file.

Inputs

  • Set the Resolution of calculations to 60.

  • Set the Maximum Simulation Time to 600 minutes to simulate a two day fire with a five hour per day burning period.

  • Change the Interval for Minimum Travel Paths to 300.

Outputs

Click the Select All button. We'll be looking at all the possible MTT outputs.

Make sure to click the Apply button and your "Run:" dialog box should look like this.

Notice the messages at the bottom, if after you click the Apply button the Launch button is not active these messages can give you a hint of the missing parts of your run.

Launch

Click the Launch button and the "FlamMap Progress..." dialog box appears.

When the run is completed, click the OK button to return to the "Run:" dialog box. Now click the OK button in the "Run:" dialog box to return to the "Project" window where you can view the outputs of the run.

Outputs

After closing the "Run:" dialog box the "Project" window will look like this.

Now expand the Basic MTT run item (by clicking the icon) in the "Tree" pane to show all the output items.

You should save the project at this point with the File > SaveAs command. Select a name for the project, don't just use the default FlamMap1 name, and remember the directory where you saved it. Then open the Evaluating MTT Outputs lesson to further explore the results of this run.
