Advanced Lesson

Evaluating MTT Outputs

This lesson gives you some tricks for viewing and interpreting MTT outputs. It uses the outputs generated in the Minimum Travel Time lesson, the four vector and 10 grid output items are all the available MTT output. You do not necessarily have to use the Schley data for this lesson, you can use outputs from a different landscape, just be sure you have all the output themes. This lesson assumes you have some experience using FlamMap to modify legends, changing units, and displaying different themes. If you do not have these skills, first complete the Basic Lessons. If you need to review these skills you can click on the blue links in this lesson as you need to, which refer to the online help system.

After completing the run, closing the "Run:" dialog box, and expanding the run item in the "Tree" pane your project window will look like this.  

As you may have noticed from the previous lesson only seven output item are available to be selected on the Minimum Travel Time tab of the "Run:" dialog box. The following seven output items are also generated for every MTT run;

  • Minimum Travel Time Ignitions

  • Rate of Spread

  • Fireline Intensity

  • Max Spread Direction

  • Elliptical Dimension a

  • Elliptical Dimension b

  • Elliptical Dimension c

The Max Spread Direction and Elliptical Dimensions outputs are designed as inputs for other applications and will not be discussed in this lesson. You can find out more on these grids by clicking on the above links.

MTT Arrival Time Grid

Select the MTT Arrival Time item so that it is the active theme in the "Display" pane.

You can now easily see the perimeter of the fire simulated. Zoom in so that the simulated fire mostly fills the "Display" pane. If you view the landscape themes at this scale you will notice the cell size of the MTT Arrival Time grid is different from the landscape themes. The cell size of the four MTT grids (MTT ROS, MTT FLI Map, MTT Arrival Time, and MTT Node Influence) is determined by the Resolution of Calculations setting you made on the Minimum Travel Time tab in the previous lesson, which is 60 meters in this example.

Turn off (by double-clicking) the lower three vector output items; MTT Flow Paths, MTT Major Paths, and MTT Arrival Contour to make viewing easier.

The red dot is the remaining Minimum Travel Time Ignitions vector item.

The default legend (View > Legend command) has the MTT Arrival Time values sorted into 20 classes and color ramped with the darkest purple cells burning early in the simulation and the white cells burning last. The black No Data cells were not "burned" in the simulation.

The values are in approximately 30 minute classes (600 minutes/20 classes) and the decimal places have been set to zero for this example.

MTT Arrival Time Contour

Display the MTT Arrival Contour item by double-clicking. This is simply a contour of the MTT Arrival Time grid.  

To view the contour properties right-click the MTT Arrival Contour item in the "Tree" pane and select the Properties command from the shortcut menu. Looking at the Contour Interval you can see it is very close to one hour. The default contour interval is the maximum MTT Arrival Time grid value divided by ten. In this case even though we specified a simulation time of 600 minutes, the maximum value is only 595 minutes due to rounding the simulation to the cells.

You can also modify the color, style, & width of the contours with this dialog box. But to change the contour interval you need to create a new contour which will be created under the Auxiliary Themes item in the "Tree" pane.

You can look at the contours as fire progression perimeters, but remember to consider the assumptions in setting the simulation time period. In this simulation the scenario simulated was a five hour burning period for each of two days.  

MTT Major Paths

Next view the MTT Major Paths vector item, this is a subset of the MTT Flow Paths vector item which is easier to view and interpret. The number of paths shown as major paths is controlled by the Interval for Minimum Travel Paths setting on the Minimum Travel Time tab.

By displaying all paths with the MTT Flow Paths vector item you can see how the simulated fire got to each of the arrival time cells (the upper left corner is the node).  

MTT ROS Grid

Displaying the MTT ROS grid reveals a random color legend which is difficult to interpret. To change the legend colors right-click on the MTT ROS item in the "Tree" pane and select the Properties command from the shortcut menu. In the "Create/Modify Legend" dialog box pick from the Color Ramp: drop down menu, light blue to dark blue is used in the following example, and click the Apply button.

Notice that the major paths tend to follow the cells with higher ROS values.

You may have already realized there are two rate of spread grids in this run, the Rate of Spread item and the MTT ROS item. Don't confuse yourself by assuming they are the same thing. View both themes and you will notice that the Rate of Spread values are calculated for the entire landscape while the MTT ROS values are only for the simulated fire area.

Before comparing individual cells for the two themes make sure they have the same units. The default units for the MTT ROS theme are meters/minute. So select the Properties command for the Rate of Spread item and make sure the units are m/min displayed with 9 decimal places.

Display the MTT ROS item and double-click one of the mid value cells within the simulated fire to bring up the "Point Attributes" dialog box. Select the Basic MTT tab to show all the output grid values for the point you double-clicked. You will see that the Rate of Spread values are usually higher than the MTT ROS values. This is because the Rate of Spread value is the maximum rate of spread for the cell while the MTT ROS value is for the spread direction for the path the simulated fire took through the cell. This is very obvious when you query cells in areas where the simulated fire was obviously backing downhill or into the south wind.

MTT FLI Map

The MTT FLI Map displays the fireline intensity only for the simulated fire. As with the MTT ROS grid discussed above there is a corresponding Fireline Intensity grid for the entire landscape which is different from the MTT FLI Map grid. The Fireline Intensity theme show the intensity if the fire was spreading in direction of maximum spread rate while MTT FLI Map shows the intensity with the spread direction of the simulated fire.

The default legend for the MTT FLI Map is skewed by the outlying values at the top of the scale. To make the legend more informative first delete the six highest valued categories and insert two new classes between the lowest value and the No Data classes, give these two new categories values of 100 and 300. Apply the yellow-orange-brown color ramp to display the example below.

MTT Node Influence Grid

The MTT Node Influence grid shows the relative importance of each cell to the spread of the fire.

In this example the darker red cells are most important to the simulated fire spreading as it did. One way to view this theme is that when the "fire" spreads through a red cell it goes on to burn more additional cells than when the "fire" spreads through a grey cell. You can see the major paths go through the most important cells, in fact the MTT Major Paths vector theme is derived from the MTT Node Influence Grid.

Finally

The MTT method of simulating fires can be confusing and difficult to grasp. However it does bring us new ways of looking at fire spread which will be important in using the Treatment Optimization Model (TOM) feature in FlamMap. So take some time to further explore MTT methods, create new ignitions on the tutorial landscape, change the MTT or weather setting for this example, and try out your own landscapes. Remember any Landscape (.LCP) File, even those created outside of FlamMap, can be used for an MTT run.
