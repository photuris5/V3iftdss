Comparing Landscapes to Develop Treatment Alternatives

All edited landscapes, resultant model run outputs, and comparison reports created in this workflow will be filed in your workspace in the same location as landscape you choose in the following step as your originating landscape.

To view both your landscape, and editing rules at the same time, use the View Split Map button at the top left of the page.

Throughout these steps we refer to different landscape (Originating, Existing, etc.). If you need a quick reference to remember what each of these refers to, see the LAbout Landscapes in Topic.

Choose an Originating Landscape

To complete this step either:

Use the Select Landscape drop-down menu to select an existing un-edited landscape from your workspace or,

Use the Create on Map option to draw a new landscape extent and extract a new Landfire landscape

 Edit Your Landscape

It is important that the landscape you use to develop and compare treatment alternatives mimics, as closely as possible, conditions as they exist on the ground. This ensures more accurate comparison results in gauging how each alternative does or does not meet objectives. The EC may need to reflect new disturbances on your landscape, like recent wildfires, fuels treatments, or windfalls.

If you had not created an Existing Landscape yet, that could also be done here, and then edited to represent treatments.

Continue this process to create as many landscapes as needed to compare. When you are finished editing and creating new landscapes, continue on to the Model Input tab to establish conditions under which the landscapes you just created will be modeled.

Establish Model Inputs

Use this tab to establish the model state by which all landscapes in your comparison session will be run and compared. If you are unsure as to what your model state should be for your comparison session, utilize My Modeling Playground to test different weather and fuel moisture conditions on your landscape(s). Creating an Auto97th Landscape Fire Behavior report in the Landscape Summary task of the Landscape Evaluation stage of the planning cycle. can also help give you a starting point for establishing model inputs.

Once a model is run in Landscape Comparison, it is permanent, there is no way to re-run that model. For this reason, testing out various modeling inputs in My Modeling Playground before proceeding with the rest of Landscape Comparison is recommended.

To complete this step:

  1. Type wind speed and wind direction in the Wind section
  2. Select Crown Fire Method and enter Foliar Moisture Content in the Crown Fire Inputs section
  3. Populate the timelag and herbaceous and woody fuel moisture percentages in the Initial Fuel Moisture section (Optional)
  4. Add rows to establish fuel moistures for individual fuel models
  5. Save Inputs

For full details about the fire behavior modeling, inputs, and outputs, visit the Fire Behavior Basic Topic.

You can adjust model inputs even after they have been saved during your session as long as you have not run a model.

Once you have run a model in the Run Model tab with saved inputs, they cannot be modified in this session.

If you are unsure what you should use for model inputs, utilize the IFTDSS Playground to test out different weather and fuel moisture condition scenarios and/or the Auto97th Landscape Fire Behavior run and reports in the Landscape Evaluation>Landscape Summary portion of the Cycle.

 Run Your Model

The Run Model tab provides a space where you can re-name and run models for landscapes you created in the Edit Landscape tab. The Landscape column in the Run Model table will list landscape “0” as selected in the Pick Landscape tab and all of its derivatives. Again, as mentioned in previous tabs, landscapes edited off of landscape “0” outside of this workflow will be available here as well. You can model and compare those landscapes just the same as if you created them within this session. The Run Name box will be auto populated with the landscape name and the type of model being run.

Currently only runs Landscape Fire Behavior but more models will be integrated soon.

Feel free to accept the default name or rename your model run. Now you can produce Landscape Fire Behavior for landscapes of your choosing by selecting Run Model after the landscapes you would like to compare. Use the refresh button to check status.

To complete this step:

  1. Click the Refresh button next to Landscape in the Run Models table to be sure all of your landscapes are listed
  2. Click Run Model for landscapes you would like to compare
  3. Click the Refresh button next to Run Status in the Run Models table to check the status of your model outputs
