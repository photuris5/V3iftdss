FTEM Roles and Privileges

  • For a user requesting FTEM access for the first time, they are auto granted Agency viewer access if their email domain is :
    • Bia.gov only is auto-granted viewer access to BIA.
    • Nps.gov only is auto-granted viewer access to NPS.
    • Fs.fed.us only is auto-granted viewer access to FS.
    • Fws.gov only is auto-granted viewer access to FWS.
    • Blm.gov only is auto-granted viewer access to BLM.
  • No user will be auto granted Agency edit access-they can request it and be auto-granted viewer privileges but an administrator has to grant a user Editor access.
  • Pre-approved email domains receive automatic access to FTEM as Agency Viewers (with access code).
  • FTEM Viewer is a role only granted to those users that select a non-core agency upon registration.
  • The Editor checkbox is not available to users that select a non-core agency upon registration.
  • FTEM Viewers require approval from FTEM Admins.
  • A User’s role can be changed to FTEM Viewer by an Admin by de-selecting all checkboxes in the Agency Roles / Requests dialogue for a given user.
  • FTEM Admins will receive account request emails and are responsible for approving account requests for the following:

    • User requests for Agency Regions that do not have an assigned Agency Admin-regardless of email domain.

    • User requests for all non-core agencies (FTEM Viewers).

  • Only users with pre-approved email domains requesting access for a core agency will receive auto approval (Viewer only) via an FTEM access code. They are only auto approved, however, if their email domain matches the core agency they select upon requesting FTEM access. All other user/role requests require Admin approval.

 

Pre-approved User Requests

1. Pre-approved user requests Viewer access for their agency (the agency that matches their email domain) and region, the edit box was not checked

In this scenario FTEM Agency Administrators will not receive a request. A Pre-approved user is automatically granted permission for the viewer role as follows:

  1. Pre-approved User requests access for a core agency and region.

  2. Pre-approved User will automatically be emailed with an activation code to gain access to FTEM as an Agency Viewer.

 3. Pre-approved user requests Editor access for their agency and region (the edit box is checked)

  1. Pre-approved User requests access for a core agency and region.

  2. Pre-approved User will automatically be emailed with an activation code to gain access to FTEM as an Agency Viewer.

  3. The FTEM Agency Admin assigned to the Pre-approved User's Agency/Region will receive an email from the system notifying them that there is a pending Editor request for the user.

  4. FTEM Agency Admin accesses the FTEM user list and filters by All (users with pre-approved email domains are automatically Agency Viewers and are never Pending), Agency Viewer, Agency, and Region to find their user.

  5. Agency Admin clicks Grant FTEM Access under the user’s name to select and validate the role they are approving.

  6. The pre-approved User will receive an email informing them that their role has been changed to Editor. Upon login, the user will see that they are now an Agency Editor in FTEM.

7. Agency Viewer with a non pre-approved email domain requests Editor role for their Agency

  1. Pre-approved User requests access for a core agency that is not affiliated with their email.

  2. User’s Viewer request is sent to the assigned FTEM Agency Admin(s) via email. If no FTEM Agency Admin is assigned to the selected Agency/Region, the request will be sent to all FTEM Admins via email.

  3. Agency Admin accesses the FTEM user list and filter by Pending, Agency, and Region to find their user.

  4. FTEM Agency Admin clicks Grant FTEM Access under the user’s name and validates the role they are approving.

  5. FTEM Agency Admin checks the Grant editor role box and clicks Grant Access in the Grant Access dialogue. The user may now enter FTEM with their requested agency affiliation and role.

  6. User receives a notification email telling them that their FTEM Privileges have been modified. They can now log in with heir requested agency affiliation and role.

 

User Requests

2. Non pre-approved User requests Viewer access for a pre-approved agency, the edit box was not checked.

  1. User requests access for a core agency and region.

  2. User’s Viewer request is sent to the assigned FTEM Agency Admin(s) via email. If no FTEM Agency Admin is assigned to the selected Agency/Region, the request will be sent to all FTEM Admins via email.

  3. FTEM Agency Admin accesses the FTEM user list and filters by Pending, Agency, and Region to find the user. Users with email domains that are not pre-approved will remain in the Pending list until they are granted (or denied) viewer privileges by an Admin.

  4. Agency Admin clicks Grant FTEM Access under the user’s name and validates the role they are approving.

  5. Agency Admin clicks Grant Access in the Grant Access dialogue and user can now enter FTEM as an Agency Viewer for their requested Agency

 4. Non pre-approved User requests Viewer access for a pre-approved agency, the edit box is checked.

  1. User requests access for a core agency and region.

  2. User’s Viewer request is sent to the assigned FTEM Agency Admin(s) via email. If no FTEM Agency Admin is assigned to the selected Agency/Region, the request will be sent to all FTEM Admins via email.

  3. Agency Admin accesses the FTEM user list and filters by Pending, Agency, and Region to find their user. Users with email domains that are not pre-approved will remain in the Pending list until they are granted viewer privileges by an Admin.

  4. FTEM Agency Admin clicks Grant FTEM Access under the user’s name and validates the role they are approving.

  5. FTEM Agency Admin checks the Grant editor role box and clicks Grant Access in the Grant Access dialogue. The user may now enter FTEM as an Agency Editor for their selected Agency.

     

  6. User receives a notification email telling them that their FTEM Privileges have been modified. They can now log in as Agency Editor.

 6. User requests Viewer access for a non-core agency and region

  1. User requests access for a non core agency and region.

  2. FTEM Admin are automatically notified of the request.

  3. FTEM Admin approve the request.

  4. The user receives an email confirming their access as and FTEM Viewer and may now access FTEM as a Viewer.
