Shapefile Save Button not functional from 's Add Shapefile Button

To permanently save a shape file so it can be saved for later use in , it must be saved using the Edit Tools button. Detailed instructions for this process are located in the Saving Shapefiles for Future Use Topic.

 Data from IFTDSS 2.0

Data and runs from IFTDSS 2.0 will not be available in this version of . This is because is built on an entirely new system relative to previous versions.

 Operational Layer Access

For security purposes, access to Operational Layers is limited to federal government employees. In the future, we will be working to streamline access to non-federal partners.

 Working between LANDFIRE and

If your using IFTDSS landscape extents to download additional data from the LANDFIRE Data Distribution site, you'll have to re-clip the landfire files to match the if you need them to match exactly to the same extent. See the Spatial Extent of and LANDFIRE Data Topic for more information.

 Navigating back and forth in the interface

When navigating back and forth within , use the application's navigation bar, buttons, and breadcrumbs do so. Avoid using your browsers 'back' and 'forward' buttons.

 Gaps or holes in uploaded shapefiles

When uploading shapefiles with multiple adjoining polygons, will automatically join the polygons into a single dataset; this process may create erroneous gaps or vertices where these polygons once touched. Visit the Shapefile Repair Topic for details on how to repair and edit shapefiles.

 Timezone

Currently, the timezone used when timestamps an output is eastern time.
